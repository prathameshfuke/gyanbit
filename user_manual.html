02. HARDWARE

The heart of GyanBit is the Raspberry Pi RP2040. This dual-core microcontroller provides the horsepower needed to drive a monochrome OLED at a smooth 30 frames per second while handling input interrupts and PWM audio.

TECHNICAL SPECIFICATIONS

RP2040 MCU
Dual-core ARM Cortex-M0+ at 133MHz. High-performance PIO state machines.
SH1106 OLED
1.3-inch monochrome display. 128x64 resolution. I2C High-speed interface (400kHz).

PINOUT MAP (GP)

GP15
Audio PWM Output
GP2 / GP3
D-Pad Up / Down
GP4 / GP5
D-Pad Left / Right
GP6 / GP7
Action Button A / B

03. ARCHITECTURE

GyanBit Studio runs a custom JavaScript-to-MicroPython transpiler. This allows you to code in the familiar language of the web (JS) while producing fast code for the console.

THE RUNTIME LIFECYCLE

01 Validation: Checks for errors in your code.
02 Transpilation: Your JS code becomes Python code.
03 Upload: The code travels through the USB cable to the console.
04 Execution: The console starts playing your game instantly!

THE 30 FPS LOOP

GyanBit runs your code 30 times every single second! This is what makes movements look smooth. In every "tick," we check buttons, move players, and draw the screen.

bit.loop(() => { // 1. Check buttons (Input) // 2. Clear previous frame // 3. Draw new pixels });

04. API GUIDE

The bit object is like a magic box of tools. You use it to tell the screen what to show and hear what the buttons are doing.

DRAWING TOOLS

bit.clear()
Clean the screen. Like washing a chalkboard!
bit.dot(x, y)
Draw one tiny point.
bit.line(x1,y1,x2,y2)
Connect two points with a line.
bit.fill(x,y,w,h)
Draw a solid color box.
bit.fillCircle(c,r)
Draw a solid circle (for balls or players).

BUTTON TOOLS

if (bit.isHeld('up')) { // Do something when Up is pressed! }

SOUND TOOLS

bit.beep(hz, ms)
Makes a "BEEP" sound. High HZ = Squeaky, Low HZ = Grumbly.

05. BOUNCING BALL

Let's make a ball that bounces around the screen! We need to remember three things: Where it is, and how fast it is moving.

1. SET THE START

let x = 64; // Left-Right position let y = 32; // Up-Down position let dx = 2; // Speed (Left-Right) let dy = 2; // Speed (Up-Down)

2. MAKE IT MOVE

bit.loop(() => { bit.clear(); // Step 1: Update position x += dx; y += dy; // Step 2: Hit a wall? Turn around! if (x <= 0 || x >= 127) dx = -dx; if (y <= 0 || y >= 63) dy = -dy; // Step 3: Draw the ball! bit.fillCircle(x, y, 4); });

06. PADDLE POWER

Now, let's make the game interactive! We are going to add a Paddle that YOU can move using the buttons. It's like having a digital tennis racket!

1. CREATE THE PADDLE

We need to tell the GyanBit where the paddle starts. Think of paddleY as the "Up and Down" position of your racket.

let paddleY = 24; // Start near the middle const PADDLE_HEIGHT = 16;

2. LISTENING TO BUTTONS

We want the paddle to move when we press the UP and DOWN buttons. We also need to make sure it doesn't float off the screen!

// Move UP if the button is held if (bit.isHeld('up') && paddleY > 0) { paddleY -= 3; } // Move DOWN if the button is held if (bit.isHeld('down') && paddleY < 64 - PADDLE_HEIGHT) { paddleY += 3; } // Draw the paddle at the left edge bit.fill(0, paddleY, 4, PADDLE_HEIGHT);
GYANBIT_HANDBOOK_06 PAGE 07

07. THE BOUNCE TEST

This is the most important part! We want to tell the ball: "If you touch my paddle, bounce back!"

THE LOGIC

We check if the ball is hitting the left side AND if it is at the same height as our paddle. If both are true... PING!

if (x <= 4 && y >= paddleY && y <= paddleY + PADDLE_HEIGHT) { dx = -dx; // Hit the paddle! Turn around! bit.beep(800, 50); // Play a fun "Ping" sound }
